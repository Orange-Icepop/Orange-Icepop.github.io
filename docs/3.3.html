Chapter3.3 Forge与Fabric的安装与模组的添加

Java版MC的诱人之处就是它能装上千种模组,本章就将讲讲如何安装模组加载器以及如何添加模组。

3.3.1 模组与版本选择

安装模组前请先确定要安装哪些模组。模组并非都是全版本的，有的甚至只有一两个版本可用，还有一些模组只支持Forge或Fabric，因此预先确定好要装哪些模组是很有必要的。我在模组推荐页面介绍了一些好的辅助类和玩法类模组，在优化指南中介绍了一些优化模组，可以参考。

选择好模组后，要注意它们的版本和加载器。PCL2在模组下载页面标明了这两项内容，可以很快的确认适用版本。一般来讲，模组如果有多个可以共存的版本，则最好选用最高的版本，因为高版本的玩法较为丰富，且更多受到持续支持，Bug更少。当然如果有个别不太重要的模组不兼容高版本，那么去掉也无妨。

3.3.2 下载与安装

虽然我们在前文中将Forge和Fabric称为模组加载器，但事实上它们是经过修改之后的游戏核心，需要在安装的时候就确定下来。建议能使用第三方启动器就直接使用启动器下载和安装，因为这两种核心的安装比较复杂。

如果你一定要用手动下载，那么请看下去，否则跳到下一个小节。

先打开Forge或fabric官网。对于Forge，点击左栏中已经预选好的版本名，然后在右侧找到最新的稳定版（一般来说最新稳定版的兼容性最好）。接下来的操作是关键！右键下载按钮，复制链接粘贴到新标签页的地址栏中。移动光标，可以看到该URL中接了一个https开头的URL，将中间https前面的部分删掉即可直接下载安装器。

不直接点链接的原因是它会跳到一个几秒的广告网站（Forge的开发者也要赚钱嘛），再跳转下载。但这个广告网站在国内访问极易卡死，因此也无法下载，唯有直接从URL中提取出源文件链接才能继续。

Fabric官网就没有这种问题。在网站点击Download，再点击下载页面中的Download for Windows（其他平台需点击下方的Universal jar），即可下载全版本的安装器。另需注意的是，Fabric需要使用Fabric API模组（官网主页中间左侧按钮）才能正常加载，这点和Forge不同。

以上两种方案下载下来的全都是安装器，因此需要打开安装器指定安装路径并进行安装才能正常使用。

3.3.3 安装Mod

安装与卸载Mod比卸载软件简单，只要将对应的.jar文件移进移出mods文件夹即可。在PCL2中，建议你打开版本隔离，避免因为模组串版本导致的错误。下述将已打开的版本隔离的PCL2启动器为例。

来到版本选择页，点击想装模组的版本右侧的小齿轮打开设置界面，点击左侧栏的模组管理，在右侧点击打开模组文件夹就是了。你也可以直接将模组的.jar文件拖动到PCL2窗口当中快速安装。

如果你手上目前没有模组，那么先选择你想装模组的版本，然后在下载=>Mod页面搜索自己想要的模组下载即可，它会自动保存到选中版本的Mod文件夹。

Forge、Fabric与NeoForge的前世今生

Forge是最老牌的模组加载器，早在MC正式版发布前就已经立项了。一开始由SpaceToad进行开发，随后Lex成为了领导者，并带着开发者共同发展着Forge生态。然而，Lex虽然是领导者，却并不善于与他人沟通。在多次内部争端后，终于在2023年，随着一次严重的公关问题，Lex与团队的其他人分道扬镳，Lex继续开发Forge，而其他人另立NeoForge项目。NeoForge在Forge的基础上进行了重新标准化以减少模组冲突，还在与Fabric靠拢，而原本的Forge模组无需过多更改就得以运行在NeoForge上（许多模组甚至使单文件同时支持Forge和NeoForge），因此NeoForge的支持率在快速攀升中。

Fabric是在MC 1.13进行大量代码重写，导致的Forge空缺期间衍生出的一大堆模组加载器之一。作为这堆加载器中最后的赢家，它相对于Forge有更多的优势。Forge本身集成了大量API，并强制修复了一些特性，这使它自由度稍低。而Fabric主打模块化和开放，将加载器与API分开放置，从而得到了开源社区的广泛支持，渐渐能与Forge分庭抗礼。它也是目前优化模组数量最多的加载器。

诚然，由于Forge的历史和基础，Fabric无法彻底取代Forge，但Fabric的开放性与理念的先进性将让它成为带领Minecraft社区前进的领头羊。让我们共同期待NeoForge与Fabric的进步吧。
